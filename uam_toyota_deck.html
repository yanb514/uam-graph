Problem Definition 03 / 24

Use Case & Key Questions

  • Target scenario: urban air mobility corridor, SF to San Jose (~80 km), modeled as a single lane at cruise altitude for the prototype.
  • Q1 — Capacity: how do UAM density and BS layout affect base-station utilization and communication latency (RTT)?
  • Q2 — Safety coupling: at what communication load does packet loss or RTT growth begin to compromise minimum safe separation margins?
  • Q3 — Mode sensitivity: how does switching pilots from automated (HOTL) to manual-control (HWTL) video streaming change BS load — and when does concurrent HWTL tip the network into overload?
  • Q4 — Shared infrastructure: can ground-user surges (stadium events, concerts) cause cascading failures on shared 5G towers even when UAM demand alone is manageable?

Transport Model 07 / 24

UAM State Representation & Dynamics

  • State per UAM: position x(t) [m along corridor] · speed v(t) [m/s] · mode flag {HOTL, HWTL} · serving BS ID · handoff flag.
  • Dynamic model: AI default point-mass, discrete-time Euler integration.
x(t+dt) = x(t) + v(t)·dt
v(t+dt) = clip( v(t) + a(t)·dt, v_min, v_max )
  • Acceleration: computed by car-following rule below; max accel/decel currently unconstrained — needs Joby aero envelope input.
  • Wrap-around: UAM reaching end of corridor is reset to origin (prototype convenience; will be replaced by hub arrival logic).

Transport Model 08 / 24

Car-Following & Separation Logic

  • Rule: simplified time-gap following (not IDM) — chosen for clarity; IDM or OVM are drop-in replacements.
T_sep = (x_leader − x_follower) / v_follower

if T_sep < T_sep_min:
   v_follower → min(v_follower, v_leader) # yield
  • Current threshold T_sep_min: 2–5 s AI placeholder
  • Known UAM guidance: reports indicate ~1-minute time gap for automated operations — 60× larger than car norms (1–2 s).
  • ⚠️ Critical gap: this parameter dominates safety classification. Must be updated after Joby input.

Communication Model 10 / 24

Network Topology & Traffic Sources

Network Topology
  • BSs modeled as nodes in a directed attribute graph G(t); edges carry RTT and link quality.
  • Placement: uniform grid along corridor, configurable count N (default 16 over 80 km ≈ 4.7 km spacing).
  • Coverage tiers: "good" ≤ 3 km · "degraded" ≤ 6 km · "none" beyond.
  • BS selection: UAM attaches to nearest BS within coverage; hysteresis (20% of good-radius) prevents ping-pong handoffs.
Traffic Sources
  • HOTL telemetry: BSM 300 B @ 10 Hz + telemetry stream = 35 kbps/UAM
  • HWTL video: live pilot video + C2 commands = +6 Mbps/UAM during episode
  • Terrestrial surge: configurable extra load per BS, time-windowed — models concurrent smartphone traffic from nearby events.
  • Total arrival rate per BS: Σ UAM rates + external load → feeds M/M/1 queue.

Communication Model 11 / 24

Load, Latency & Handoff Model

  • BS utilization: ρ = arrival_rate / capacity (M/M/1 utilization ratio).
W_queue = L_min / (1 − ρ) # M/M/1 mean waiting time; → ∞ as ρ → 1
RTT = 2 × (d/c + W_queue + 0.01) + 0.1 if in_handoff
  • Drop model: messages exceeding buffer capacity (buffer_capacity_msgs) are dropped; handoff period adds 10% drop probability.
  • Threshold zones: ρ > 0.60 → Warning · ρ > 0.85 → Critical
  • Handoff: triggered when UAM exits hysteresis zone; 200 ms delay + 10% drop; new BS edge created in graph.
  • Planned refinements: realistic channel model (pathloss / shadowing) · Toyota / Joby 5G capacity specs · scheduling model beyond M/M/1.

Control Modes 12 / 24

Human-over-loop vs. Human-within-loop

HOTL — Human Over the Loop
  • Automation manages flight; human monitors only
  • Low bandwidth: 35 kbps telemetry stream
  • Default operating mode in simulation
  • Baseline ρ per BS ≈ 3% (manageable)
HWTL — Human Within the Loop
  • Human actively pilots via live video feed
  • High bandwidth: +6 Mbps video + commands
  • Triggered by Poisson process or zone-based rule
  • Duration: exponential(mean_duration_s)
Mode Switching Logic
  • Poisson model: each UAM switches HOTL→HWTL at rate λ/hr; episode duration ∼ Exp(mean_hwtl_s); returns to HOTL after episode.
  • Zone-based trigger: switch probability ↑ near departure/arrival hubs (configurable, not active in baseline).
  • Parameters: λ, mean_hwtl_duration_s, max_concurrent_hwtl (all configurable per scenario).

Control Modes 13 / 24

Mode Impact on Communication & Safety

  • Single HWTL event on a 20 Mbps BS: load jumps from ~3% → ~33% per HWTL aircraft. Two concurrent HWTL events → 63% (warning zone); three → 93% (critical zone).
  • Non-linear M/M/1 effect: as ρ → 1, queue delay W = L/(1−ρ) grows without bound — RTT can reach seconds even with moderate UAM counts.
  • Safety coupling: RTT > 500 ms → pilot commands may expire before delivery → HWTL becomes unsafe; simulation flags this as UNSAFE.
  • Classification impact: drop rate > 5% OR any BS spending > 10% of run time in critical zone → DEGRADED; separation violation OR RTT > 500 ms → UNSAFE.

Scenario 2 19 / 24

High Density — 150 UAMs

Configuration (changed params)
  • 150 UAMs (3×) · 2 lanes
  • 16 BS × 50 Mbps (halved)
  • T_sep_min = 3 s · seed 123
Root Cause
  • BS16 (near SJ hub) serves 56 UAMs at peak vs. ~9 by design — geographic bunching near destination.
  • 56 UAMs × 20 msgs/step = 1,120 msgs/step >> 300-msg buffer → 73% local drop rate.
  • Aircraft bunching → headway shrinks to 1.5 s < 2.0 s threshold.
Results
  • ✗ Min separation: 1.5 s < 2.0 s
  • ✓ Max RTT: 134 ms (OK — bandwidth not saturated)
  • ✗ Drop rate: 13% > 5%
  • ✓ Peak BS load: 29% bandwidth (but buffer overflow)
  • ✗ Sep violations: 3,113 steps
✗ UNSAFE — Buffer overflow + separation
  • Key insight: bandwidth utilization looks healthy (29%) while the system is actually failing — bottleneck is the message buffer, not bandwidth. Diagnosis requires looking at both metrics.

Scenario 3 20 / 24

Frequent Switches — λ = 5/UAM/hr

  • Config change: 80 UAMs · λ = 5 switches/UAM/hr (10× baseline) · mean HWTL duration = 45 s · seed 777.
  • Expected load: E[concurrent HWTL/BS] = (5/3600) × 45 × (80/16) ≈ 3 → adds ~18 Mbps to each 100 Mbps BS → ρ ≈ 18% additional → total ρ ≈ 21% (warning zone never reached).
Results
  • ✓ Min separation: 3.3 s
  • ✓ Max RTT: 132 ms
  • ✓ Drop rate: 1.42%
  • ✓ Peak BS load: < 30%
  • Mode switches: 96 (all absorbed)
✓ NORMAL — Infrastructure sized for HWTL
Why it passes
  • Pilot takeovers are frequent but the corridor is not crowded — 5 UAMs/BS average.
  • 100 Mbps BS has headroom specifically for occasional HWTL video bursts.
  • Demonstrates that high switch rate alone is not unsafe — it is the combination with insufficient infrastructure that matters.

Scenario 4 21 / 24

Cascade Failure — Terrestrial Surge

  • Config: 60 UAMs · 6 BS × 20 Mbps · λ = 8/hr · mean HWTL = 90 s · terrestrial surge +8 Mbps/BS at t = 120–360 s.
  • Math: E[concurrent HWTL/BS] = (8/3600)×90×10 ≈ 2 → 2×6 Mbps = 12 Mbps → ρ = 60%. Add +8 Mbps → 20 Mbps → ρ = 100% → M/M/1 saturated.
